\section{OTA / tdconnect, Visual Basic Script}
Die Open Test Architecture API (OTA) erlaubt es mit tdconnect Zugriff auf HP
Application Lifecycle Manager über externe Scripte zu haben. Die Schnittstelle
wird mit Visual Basic Script angesprochen. Für diesen Zweck wurde Visual Basic
Script C(sharp) bevorzugt da diese Sprache für HPQC am geeignetsten ist. VBS ist
gut dokumentiert und verfügt über eine viel größere Bibliothek was die Steuerung des
Tools anbelangt. Der Workflow von HPQC ist auch mit VBS beschrieben, was eine
Andockung mit der selbigen vereinfacht. C(sharp) scheint zu diesen Zeitpunkt
noch nicht für die Software ausgearbeitet genug zu sein. Das kann sich in Zukunft vielleicht ändern.
